# CS・ロボット論文 — 執筆・レビューチェックリスト

Used by `peer_reviewer_agent`, `draft_writer_agent` (self-review step).

対応する節の流れ → `cs_paper_section_flow.md`
対応するルール集 → `cs_paper_writing_rules.md`

**使用上の注意:** 以下の項目はすべて必須ではない。研究内容によっては一部に答えられない場合や、明示的に書く必要がない場合がある。論文の主張を支えるうえで本当に必要な情報かどうかを判断してから確認する。

---

## Part A — 執筆時チェックリスト（各章）

### A-1: Introduction

- [ ] 本研究の社会的背景は何か。
- [ ] 第1文は、読者と共有できる最大公約数的な事実から始まっているか。
- [ ] 本研究の target task / target problem は何か。
- [ ] target problem の具体例やユースケースは何か。
- [ ] その target problem はなぜ難しいのか。
- [ ] 既存手法が誤る例はどのようなケースか。
- [ ] 既存手法群はなぜ不十分なのか（個別論文ではなく研究群として）。
- [ ] 本研究では何を提案し、何を解決するのか。
- [ ] 提案手法は既存手法と何が違うのか。
- [ ] 既存手法と異なる部分をなぜ導入するべきなのか。
- [ ] 提案手法の新規性は何か。
- [ ] 提案手法全体の構成を示す eye-catch figure は必要か。

### A-2: Related Work

- [ ] 関連分野のサーベイ論文を複数把握しているか。
- [ ] 関連する主要な研究分野（複数）は何か。
- [ ] 標準データセットが存在する場合、それは何か。
- [ ] 提案手法と類似手法との差分は何か。
- [ ] 先行研究の羅列ではなく、本研究の位置づけが自然に伝わるか。

### A-3: Problem Setting / Problem Statement

- [ ] 対象とするタスクの名称と内容は何か。
- [ ] 対象タスクの望ましい解や出力は何か。
- [ ] 何をもって良い解とするのか。
- [ ] 対象タスクの代表例は示されているか。
- [ ] このタスクで与えられる入力は何か。
- [ ] タスクで求められる出力は何か。
- [ ] 使用する用語は定義されているか。
- [ ] 本研究では何を扱わないのか。
- [ ] 本研究は何を前提にしているのか。
- [ ] タスクの評価尺度は何か。

### A-4: Proposed Method / Method

- [ ] 本研究は何の手法を拡張したものか。
- [ ] 提案した拡張は、他の手法にも広く適用可能か。
- [ ] 提案手法と既存手法の違いは何か。
- [ ] 提案手法はいくつの主要モジュールを持つか。
- [ ] 各主要モジュールの名称と役割は何か。
- [ ] 提案手法のモデル構造は図で示されているか。
- [ ] 入力は記号や数式で定義されているか。
- [ ] 各入力の意味や次元は説明されているか。
- [ ] 入力特徴はどのように抽出されるか。
- [ ] 各モジュールの motivation、役割、入出力、構造は説明されているか。
- [ ] 予測は数式で定義されているか。
- [ ] 損失関数は定義されているか。
- [ ] 代替手法や設計選択がある場合、その採用理由は説明されているか。

### A-5: Experimental Setup

- [ ] どのデータセットを使用したか。
- [ ] 新規データセットを構築した場合、どのように構築したか。
- [ ] アノテーション方法や指示内容は説明されているか。
- [ ] 標準データセットを使った、または使わなかった理由は何か。
- [ ] データセットの前処理やデータ拡張は説明されているか。
- [ ] データセットの統計情報は示されているか。
- [ ] training / validation / test set の分割方法とサイズは示されているか。
- [ ] 各分割をどのように使用したか。
- [ ] 最適化手法、エポック数、ハイパーパラメータは示されているか。
- [ ] パラメータ数や計算量は必要に応じて示されているか。
- [ ] ハードウェア構成は示されているか。
- [ ] 訓練時間や推論時間は必要に応じて示されているか。

### A-6: Results / Experimental Results

- [ ] ベースライン手法との定量的比較結果は示されているか。
- [ ] 何をベースラインとしたか。
- [ ] なぜそのベースラインを選んだのか。
- [ ] 評価尺度は数式や本文で説明されているか。
- [ ] 複数の評価尺度がある場合、主要尺度は明確か。
- [ ] なぜその評価尺度を使用したのか。
- [ ] 性能は相対値だけでなく絶対値で示されているか。
- [ ] 統計的有意性を主張する場合、検定結果は示されているか。
- [ ] 成功例は Ground Truth、ベースライン、提案手法の出力とともに示されているか。
- [ ] 成功例について、なぜ適切だったのかを説明しているか。
- [ ] 失敗例は示されているか。
- [ ] 失敗例について、なぜ失敗したのかを説明しているか。
- [ ] Ablation study では、何のために何を取り除いたかが説明されているか。
- [ ] Ablation の結果から、取り除いた要素の有効性が説明されているか。
- [ ] 混同行列は必要に応じて示されているか。
- [ ] 失敗例の総数は示されているか。
- [ ] 失敗例はカテゴリに分類され、各カテゴリの定義が示されているか。
- [ ] 失敗の主要因と possible solution は説明されているか。

### A-7: Conclusion

- [ ] 本研究ではどのようなタスクを扱ったか。
- [ ] 本研究では何を提案したか。
- [ ] 本研究の貢献は過去形で簡潔にまとめられているか。
- [ ] 実験によって何が明らかになったか。
- [ ] 将来研究は何か。
- [ ] 本文で支えられていない強い主張をしていないか。

## Part B — レビュー時チェックリスト

### B-1: 全体構成

- [ ] 研究目的が早い段階で明確になっているか。
- [ ] 目的、問題、提案、実験、結果、考察が一貫しているか。
- [ ] 章ごとの内容が孤立していないか。
- [ ] Introduction と Conclusion が対応しているか。
- [ ] Related Work が本研究の位置づけを明確にしているか。
- [ ] Problem Setting と Method の接続が自然か。
- [ ] Method と Experiments の接続が自然か。
- [ ] Results と Discussion の役割が混同されていないか。

### B-2: Introduction

- [ ] 背景から目的へ自然につながっているか。
- [ ] 第1文が読者と共有しやすい事実から始まっているか。
- [ ] 目的を述べる前に既存研究の詳細に入りすぎていないか。
- [ ] target problem が明確か。
- [ ] 典型的なユースケースが必要に応じて示されているか。
- [ ] 問題が「何ができないのか」として具体的に書かれているか。
- [ ] 既存手法群の不十分さが抽象化されているか（1つの論文への依存がないか）。
- [ ] アイデア、実装方法、評価が混同されていないか。
- [ ] 提案手法の差分と、その差分を導入する理由が説明されているか。
- [ ] 貢献が「作ったもの」だけでなく「示したこと」まで含んでいるか。

### B-3: Related Work

- [ ] 先行研究の羅列になっていないか。
- [ ] 本研究の目的と関連する流れで整理されているか。
- [ ] 既存研究との差分が自然に伝わるか。
- [ ] 重要な研究群やサーベイが抜けていないか。
- [ ] 標準データセットや標準的評価との関係が必要に応じて説明されているか。
- [ ] 引用が目的に対して過不足ないか。

### B-4: Problem Setting

- [ ] 前提、入力、出力、目的が明確か。
- [ ] 望ましい解や成功条件が説明されているか。
- [ ] 実験固有の条件が混ざっていないか。
- [ ] 初出の用語が自然に定義されているか。
- [ ] Method に必要な記号や概念が先に説明されているか。
- [ ] 本研究で扱わない範囲が必要に応じて明確になっているか。

### B-5: Proposed Method

- [ ] 冒頭で解決方針が説明されているか。
- [ ] 手法全体の overview があるか。
- [ ] 各節の役割が明確か。
- [ ] 節タイトルが処理の役割を表しているか。
- [ ] 主要モジュールの動機、役割、入出力、構造が説明されているか。
- [ ] 既存部分と提案部分が区別されているか。
- [ ] 入力、予測、損失関数が明確に定義されているか。
- [ ] 数式やアルゴリズムが本文の流れに沿って導入されているか。
- [ ] 一般性を主張する場合、その根拠があるか。

### B-6: Experiments / Results / Discussion

- [ ] 実験が主張を検証する構成になっているか。
- [ ] 比較手法と評価指標が妥当か。
- [ ] ベースラインの選定理由が説明されているか。
- [ ] データセットや実験条件が再現可能な程度に説明されているか。
- [ ] 結果から何がわかったのかが書かれているか。
- [ ] 定量結果の表が本文で解釈されているか。
- [ ] 定性的結果が解釈つきで説明されているか。
- [ ] 性能低下や失敗例の原因が切り分けられているか。
- [ ] Ablation study の目的と結果の意味が明確か。
- [ ] 限界と改善方針が示されているか。

### B-7: 日本語表現（日本語で執筆する場合）

- [ ] 一文が長すぎないか。
- [ ] 係受けが遠くないか。
- [ ] 独自用語が未定義で使われていないか。
- [ ] 「〜が必要である」「〜が重要である」だけで終わる文が多くないか。
- [ ] 箇条書きのメモを本文にしたような文章になっていないか。
- [ ] 「技術的問題点として」「キーアイデアとして」など、内部整理語が本文に出ていないか。
- [ ] 段落内・段落間の流れが自然か。
- [ ] 同じ文型の繰り返しが多すぎないか。
